Build Module (Factory System)

Purpose: frictionless production of brand + website + assets using strict gates, parallel execution, and one final QA standard. Clients do one 30-min direction call only — then Build executes.

Build Flow (from Gate to Delivery)

Build is not allowed to “ask questions”. Build executes pre-approved inputs. Anything outside scope becomes a new order.

Phase 0 — Build Entry Gate

Owner: Nana

Build does not start unless all required inputs exist.

  • Package confirmed + scope locked
  • Payment received (or invoice approved)
  • Client inputs submitted (forms, access, info)
  • Brand Direction Call booked (30 min)

Phase 1 — Brand Direction Lock (30 min)

Owner: Ammar

One meeting only. No ideation. Output is a clear direction sheet.

  • Brand feeling (bold / calm / premium / playful)
  • Color direction (2–3 options max)
  • Visual references (approved)
  • Positioning (what we are / not)

Phase 2 — Parallel Execution

Owners: Elias + Ammar

Website starts immediately with provisional colors while branding is assembled using AI tools.

Lane A — Website Pre-Build Elias

Build fast using Lovable/Antigravity templates (React). No custom layouts.

  • Input: Brand Direction Sheet + package type
  • Actions: Structure + sections + provisional colors
  • Output: Website skeleton ready for final brand assets

Lane B — Branding Assembly Ammar

Factory branding only. AI-assisted mockups (Nano Banana) + human refinement.

  • Tools: Nano Banana (mockups) + AI (variations)
  • Outputs: Logo + palette + typography + 1–2 mockups
  • Rule: No endless concept loops

Phase 3 — Merge & Polish

Owner: Ammar

Inject final brand assets into website and align visuals. Minor adjustments only.

  • Merge: Final logo, colors, typography into site
  • Polish: Spacing, consistency, assets, visuals
  • Limit: No structural changes

Phase 4 — QA & Packaging

Owner: Ammar

One quality gate. If Ammar says “yes” → Nana delivers.

  • QA: Mobile check + final visual review
  • Packaging: Brand files + access + summary
  • Ready: “What you got” delivery pack

Phase 5 — Delivery & Account Ownership

Owner: Nana

Client walkthrough + scope control + upsell path (tech support / accounting).

  • Delivery: Walkthrough + credentials handover
  • Scope: Extra requests become new order
  • Next: Tech support + accounting anchor

Factory Laws (Carved in Stone)

NO AGENCY BEHAVIOR

Build Must Be Boring (and fast).

If Build is asking the client questions mid-project, the system is broken. Build executes locked instructions — upgrades are sold separately.

  • No Premium Branding in Factory Branding is always bounded and repeatable.
  • One Client Touchpoint Only 30-min Brand Direction Call, then execution begins.
  • No Missing Inputs If client inputs are missing, Build does not start.
  • Parallel Execution Is Mandatory Website starts immediately; branding runs in parallel.
  • Ammar Is Final QA Gate One eye, one standard, one “yes/no”.
  • Nana Owns the Client After Build Delivery, follow-ups, scope control, and upsells.
Key tools: Lovable/Antigravity (web assembly) · Nano Banana (mockups) · AI-assisted variants · strict checklists · no custom scope.
