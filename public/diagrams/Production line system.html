Production Line System (Carved in Stone)

Every client moves through the same factory flow. Stages (1/2/3) change what modules activate in the Build step — not the core flow. There is one client handover only: at the end.

1) Intake & Build Map

Purpose: Decide exactly what gets built (no work starts yet).

Gate 0

Outputs

  • Purchased stage confirmed (1 / 2 / 3)
  • Company type (EF / AB)
  • Complexity flags (payroll, EU sales, integrations)
  • Build Map: Modules ON / OFF / UPGRADED
Rule: No production starts before the Build Map exists.

2) Formation (Backend Only)

Purpose: Make the company legally alive + accounting system created.

Stage 1 Core

Always included

  • Company registration
  • F-skatt / VAT (if applicable)
  • Accounting system setup
Important: Formation is not delivered to the client. It is internal.

3) Build (Modules Activate)

Purpose: Build only what is required to reach the purchased readiness stage.

Stage 1/2/3 Logic

Key principle

  • Modules are ON / OFF / UPGRADED by the Build Map
  • Higher stages replace lower-resolution modules
  • Example: Stage 2 website replaces Stage 1 landing page
Rule: Modules override — they never stack or duplicate.

4) Handover (Only Client Exit)

Purpose: Deliver the finished company + lock ongoing accounting.

Factory Exit

Handover checklist

  • Deliverables verified (exit criteria met)
  • Access transferred (systems & accounts)
  • Accounting plan confirmed + autopay set
  • Client responsibilities explained
  • Production ticket closed
Golden Rule: There is one handover only per production cycle.

✅ What We Do (Always)

Follow the Build Map Only modules marked ON are built. Upgrades replace lower versions.
One finished delivery We deliver at handover. We do not show unfinished work in production.
Protect boundaries If it’s not listed, it’s not included. Scope changes require a change order.
Improve between batches We fix the system after delivery — not during client production.

❌ What We Never Do

No partial handovers No “quick delivery” of parts. No mid-production client reviews.
No exceptions inside production Exceptions are logged for system improvement — never executed.
No stacking modules Stage 2 does not include Stage 1 landing page. Stage 3 does not include Stage 2 website if replaced.
No selling custom scope Sales cannot modify stages. If needs are different, it becomes a new order.
One-line definition: Spikad runs a factory flow: Intake → Formation → Build → Handover. Stages (1/2/3) change which modules activate in Build. There is one client handover only.
